Библиотека предоставляет декларативный способ валидации HTML-форм с использованием нативных браузерных API валидации и ARIA атрибутов для доступности.

## Требования
Для корректной работы формы должны иметь:
Структура формы:
- Все поля ввода должны иметь атрибут name
- Каждое поле должно иметь связанный элемент 'label'
- Каждое поле должно иметь атрибут aria-describedby, указывающий на ID элемента для вывода ошибок

Элементы вывода ошибок:
- Элемент с ID, соответствующим значению aria-describedby поля ввода
- Обычно используется элемент 'output', 'span' или 'div'

## Установка и использование
Импортируется один единственный раз в начале файла с помощью 'import'

### Базовый пример
```typescript
const myForm = document.querySelector('#userForm') as HTMLFormElement; // Получаем форму

const validator = formValidator(myForm); // Инициализируем валидатор

// Настраиваем правила валидации для полей

validator
  .field('username')
  .string()                    // Только буквы
  .min('Минимум 3 символа')   // Минимальная длина
  .max('Максимум 20 символов'); // Максимальная длина

validator
  .field('age')
  .number()                   // Только цифры
  .min('Минимальный возраст: 18')
  .max('Максимальный возраст: 100');

validator.validate();// Активируем валидацию
```
## API
### form(element: HTMLFormElement)**
Основная функция, создающая экземпляр валидатора.

Параметры:

- element - HTML элемент формы

    Возвращает: объект валидатора с методами field() и validate()

Исключения:

- Error("Форма не передана") - если элемент не передан

- Error("В форме нет инпутов") - если в форме нет полей ввода

- Error("У поля нет своего лейбла") - если у поля нет связанного <label>

- Error("У инпута нет имени") - если у поля нет атрибута name

- Error("Отсутствует поле ввода с атрибутом aria-describedby") - если нет атрибута aria-describedby

- Error("Нет output") - если элемент с ID из aria-describedby не найден

### validator.field(fieldName: string)
Начинает цепочку валидации для конкретного поля.

Параметры:

- fieldName - значение атрибута name поля ввода

    Возвращает: объект с методами валидации

Исключения:

- Error("Поле {fieldName} не найдено") - если поле с указанным именем не существует

### Методы валидации
Все методы возвращают тот же объект методов, позволяя создавать цепочки вызовов.

- .min(message: string)
    
    Устанавливает сообщение об ошибке для минимального значения/длины.
    
    Использует нативное браузерное ограничение minlength для текстовых полей и min для числовых.

- .max(message: string)
    
    Устанавливает сообщение об ошибке для максимального значения/длины.
    
    Использует нативное браузерное ограничение maxlength для текстовых полей и max для числовых.

- .string()

    Проверяет, что поле содержит только буквы (латинские и кириллические).
    
    Устанавливает паттерн: [a-zA-Zа-яА-Я]+

    Исключения:

    - Error("В поле должен быть текст") - если поле не имеет тип text

- .number()
    
    Проверяет, что поле содержит только цифры.
    
    Устанавливает паттерн: [0-9]+

### validator.validate()

Активирует валидацию формы. Добавляет обработчик события submit к форме.

Поведение при валидации:

- Предотвращает стандартную отправку формы

- Проверяет валидность всех полей

- При ошибках:

    Выводит сообщение "Ошибка" в связанный элемент

    Устанавливает aria-invalid="true" для поля

- При успешной валидации:

    Очищает атрибут aria-invalid

    Сбрасывает форму

## Полнота и понятность
### Преимущества:
- Использует ARIA атрибуты для скринридеров

- Использует браузерные API (checkValidity())

- Полная поддержка TypeScript

### Ограничения:
- Поддерживает только минимальные/максимальные значения и паттерны

- Только синхронная проверка

- Требует строгой структуры HTML
